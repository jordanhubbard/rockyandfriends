## 2. Rust Mapping

### New crate: `ccc-server` (the big one)

Replaces: `api/index.mjs` + all `api/routes/*.mjs`

This becomes a full Axum server alongside the existing `dashboard-server`. Options:
- **Option A**: Merge into `dashboard-server` (single binary, one port)

## 5. Risks & Gotchas

### ClawBus SSE (HIGH RISK)
The bus is a long-lived SSE stream with multiple concurrent subscribers. Node's event-loop handles this naturally. In Rust:
- Use `tokio::sync::broadcast::channel` as the in-memory pub/sub
- Wrap in `axum::response::sse::Sse` with `tokio_stream::wrappers::BroadcastStream`
- Must handle slow consumers (broadcast drops old messages — size the buffer appropriately, currently need to look at `bus.jsonl` append behavior)
- The `bus.jsonl` + `acks.jsonl` + `dead-letter.jsonl` pattern needs to be preserved for persistence

### Vector Client (Qdrant — migration complete)
Milvus has been retired and replaced by Qdrant (port 6333). The Qdrant REST API is used via reqwest.

### HMAC Auth (LOW RISK)
`timingSafeEqual` for token comparison. Rust's `subtle` crate handles this. The multi-token `AUTH_TOKENS` set maps cleanly to a `HashSet<String>` + constant-time compare.

### File-based state (LOW RISK)
Queue, agents, secrets, brain-state all persist as JSON files. `serde_json` + `tokio::fs` handles this. Locking: Node uses sequential async writes (safe-ish). Rust should use `tokio::sync::RwLock<State>` with periodic flush, not per-request writes.

### SSE + file tailing for bus replay (MEDIUM RISK)
New SSE subscribers get replayed recent messages from `bus.jsonl`. Need to replicate the tail-and-follow behavior. Use `tokio::fs::File` + manual seek + `tokio::time::interval` polling, or `notify` crate for inotify-based watching.

### Exec sandbox (LOW RISK)
`exec/agent-listener.mjs` runs arbitrary shell commands with an allowlist. In Rust: `tokio::process::Command` + allowlist check before spawn. No VM sandbox equivalent needed — the allowlist approach is the same.

### Legacy HTML routes (LOW PRIORITY)
`api/routes/ui.mjs` renders HTML pages. These are largely superseded by the Leptos WASM dashboard. Recommend: stub them with redirects to the WASM app, don't port the templates.
