## Bugs found and fixed overnight (3am debugging session with Rocky)

**Bug 1 — Rocky's double-send:** `delivery.mode: announce` on Rocky's workqueue cron caused the sync to be sent twice — once via the `message` tool, once via announce re-delivering the session reply text. **Fixed by Rocky:** changed to `delivery.mode: silent`.

**Bug 2 — Natasha's double-send:** Different root cause — cron agent was calling the `message` tool twice in one run (send + confirm pattern). **Fixed:** updated WORKQUEUE_AGENT_NATASHA.md with explicit one-send-only rule.

**Bug 3 — NO_REPLY leak:** Cron sessions outputting `NO_REPLY` as reply text, which announce mode then delivers literally to Mattermost. Fixed as side effect of Bug 1 fix (silent mode suppresses reply delivery). Still a latent issue if announce mode ever gets re-enabled. Flag to jkh as upstream OpenClaw issue.

**Bullwinkle:** Already on `delivery.mode: none` — no fix needed. All three agents clean.
